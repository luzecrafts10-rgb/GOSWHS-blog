## 🚀 GitHub Pages 배포 방법 (단계별)

### 1단계 — GitHub 저장소 만들기

1. [github.com](https://github.com) 에 로그인 (없으면 가입)
2. 우상단 **`+`** → **New repository** 클릭
3. Repository name: `swhs-blog` (원하는 이름)
4. **Public** 선택
5. **Create repository** 클릭

### 2단계 — 파일 올리기

**방법 A — 웹 브라우저 (간단)**

1. 저장소 페이지에서 **Add file** → **Upload files** 클릭
2. `index.html`, `style.css`, `app.js` 세 파일을 드래그 업로드
3. 아래 Commit changes 버튼 클릭

**방법 B — Git 사용 (권장)**

```bash
git init
git add .
git commit -m "초기 배포"
git branch -M main
git remote add origin https://github.com/[내아이디]/swhs-blog.git
git push -u origin main
```

### 3단계 — GitHub Pages 활성화

1. 저장소 → **Settings** 탭 클릭
2. 왼쪽 메뉴 **Pages** 클릭
3. Source: **Deploy from a branch**
4. Branch: **main** / `/ (root)` 선택
5. **Save** 클릭
6. 약 1~2분 후 `https://[내아이디].github.io/swhs-blog` 로 접속 가능

## 🔑 관리자 모드 사용법

1. 로그인 화면에서 아이디/비밀번호 입력 후
2. **관리자 키** 입력란에 아래 키를 입력:

```
sw.admin.26
```

3. 로그인하면 상단 네비게이션에 **관리자** 메뉴가 나타남
4. 관리자 패널에서 가능한 작업:
   - 전체 게시글 조회·숨김·복원·삭제
   - 전체 회원 조회·탈퇴처리
   - 신고 내역 확인·처리

## 📱 휴대폰 인증 안내

현재는 **시뮬레이션 방식**입니다. 인증번호 발송 버튼을 누르면 팝업창에 번호가 표시됩니다.

실제 SMS 인증을 원하신다면 다음 서비스 중 하나를 연동해야 합니다:
- **Naver Cloud Platform SMS** (국내 추천)
- **Twilio** (해외)
- **알리고** (저렴한 국내 옵션)

단, 이를 위해서는 서버(백엔드)가 필요하므로 GitHub Pages만으로는 불가합니다.  
백엔드가 필요할 경우 **Firebase** 또는 **Supabase** 연동을 추천드립니다.

- 데이터는 브라우저 **localStorage**에 저장됩니다
